Guide — Modifications Apps Script & Next.js

Détail de toutes les modifications apportées pour automatiser la génération des liens d'activation et l'envoi des emails de réinitialisation.

Apps Script — 5 modifications Next.js — 1 modification Variable {{activation_link}}
🔁

Vue d'ensemble — 2 flux automatisés

Les modifications permettent deux choses : (1) envoyer automatiquement le lien d'accès espace client après paiement, (2) envoyer le lien de réinitialisation quand un client a oublié son mot de passe.

Flux 1 — Accès espace client (après paiement)
CRM Colonne is_paiement_recu cochée à oui
↓
Apps Script maybeSendIdentifiantEmail() appelle l'API pour générer un lien (2h)
↓
API /api/admin/clients/generate-link crée un token sécurisé dans Supabase
↓
Email Template envoi_identifiant_id envoyé avec {{activation_link}} rempli
↓
Site Client clique le lien → page /connexion/activer → définit son mot de passe
Flux 2 — Mot de passe oublié
Site Client saisit son email sur /connexion/mot-de-passe-oublie
↓
Next.js Génère un lien sécurisé (2h) via /api/admin/clients/generate-link
↓
Apps Script doPost() reçoit l'email + le lien et envoie le mail de réinitialisation
↓
Email Template reset_mot_de_passe envoyé avec {{activation_link}} rempli
📜

Modifications Apps Script — 5 changements

1
setConfig() — Ajout de APP_URL
Ligne ~85 · 1 ligne ajoutée

Ajoute l'URL du site dans les propriétés du script, pour que les autres fonctions puissent appeler l'API Next.js.

properties.setProperties({
  API_URL: "https://optimal-logic.com/api/admin/prospects-sync",
  API_SECRET: "TON_SECRET",
  ...
  APP_URL: "https://optimal-logic.com",  // ← AJOUTÉ
});
⚠️ Après avoir collé le code, exécuter setConfig() une fois dans l'éditeur Apps Script pour enregistrer cette nouvelle propriété.
2
getConfig() — Lecture de APP_URL
Ligne ~100 · 1 ligne ajoutée

Expose la valeur APP_URL dans l'objet config retourné, pour qu'elle soit accessible partout dans le script.

return {
  apiUrl:        properties.getProperty("API_URL"),
  apiSecret:     properties.getProperty("API_SECRET"),
  ...
  appUrl:        properties.getProperty("APP_URL"),  // ← AJOUTÉ
};
3
replaceVariables() — Variable {{activation_link}}
Ligne ~278 · 1 ligne ajoutée

Permet aux templates email d'utiliser {{activation_link}} pour insérer automatiquement le lien d'activation.

return String(text)
  .replaceAll("{{prenom}}", row.prenom || "")
  ...
  .replaceAll("{{date_heure_rdv}}", row.dateHeureRdv || "")
  .replaceAll("{{activation_link}}", row.activation_link || "");  // ← AJOUTÉ
📝 Dans vos templates Google Sheets, utilisez {{activation_link}} pour insérer le lien dans les templates envoi_identifiant_id et reset_mot_de_passe.
4
maybeSendIdentifiantEmail() — Génération automatique du lien
Ligne ~396 · ~25 lignes modifiées

Avant d'envoyer l'email d'identifiants, la fonction appelle maintenant l'API pour générer le lien d'activation. Si l'API échoue, l'envoi est annulé et l'erreur est loguée.

function maybeSendIdentifiantEmail(sheet, row, rowNumber) {
  if (!isTruthy(row[HEADERS.isPaiementRecu])) return;
  if (row[HEADERS.confirmationPaiementEnvoyeeAt])) return;
  if (row[HEADERS.identifiantRecuAt]) return;

  // Appel API pour générer le lien sécurisé (2h)
  const config = getConfig();
  try {
    const response = UrlFetchApp.fetch(
      `${config.appUrl}/api/admin/clients/generate-link`,
      {
        method: "post",
        headers: {
          "Content-Type": "application/json",
          "x-admin-secret": config.apiSecret,
        },
        payload: JSON.stringify({ email: row.email }),
        muteHttpExceptions: true,
      }
    );
    const data = JSON.parse(response.getContentText());
    if (!data.link) throw new Error("Lien non généré");
    row.activation_link = data.link;  // injecté dans replaceVariables
  } catch (err) {
    logEmail({ ..., statut: "erreur", erreur: "Génération lien: " + err.message });
    return;  // on n'envoie pas si le lien n'a pas pu être généré
  }

  sendTemplateEmail(row, "envoi_identifiant_id");
  updateCellByHeader(sheet, rowNumber, HEADERS.identifiantRecuAt, new Date());
}
5
doPost() — NOUVELLE fonction
Fin du fichier · ~30 lignes ajoutées

Fonction entièrement nouvelle. Apps Script devient un endpoint web capable de recevoir des requêtes POST depuis le site. Il reçoit l'email du client + le lien déjà généré, et envoie le mail de réinitialisation.

function doPost(e) {
  try {
    const data = JSON.parse(e.postData.contents);
    const config = getConfig();

    // Vérification du secret pour sécuriser l'endpoint
    if (data.secret !== config.apiSecret) {
      return ContentService
        .createTextOutput(JSON.stringify({ error: "Non autorisé" }))
        .setMimeType(ContentService.MimeType.JSON);
    }

    const row = {
      email: data.email,
      prenom: data.prenom || "",
      nom:    data.nom    || "",
      activation_link: data.link,
      demande_id: "reset_mdp",
    };

    sendTemplateEmail(row, "reset_mot_de_passe");

    return ContentService
      .createTextOutput(JSON.stringify({ success: true }))
      .setMimeType(ContentService.MimeType.JSON);

  } catch (err) {
    return ContentService
      .createTextOutput(JSON.stringify({ error: err.message }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
⚠️ Pour activer doPost, il faut redéployer le Web App depuis Apps Script → Déployer → Gérer les déploiements → Nouvelle version. Puis copier l'URL du Web App dans la variable d'environnement APPS_SCRIPT_URL sur Vercel.

Modification Next.js — 1 changement

1
requestPasswordReset() dans actions.ts
app/(main)/connexion/actions.ts · fonction remplacée

L'ancienne version utilisait le système d'email de Supabase (qui expirait rapidement et était limité à 3 mails/heure). La nouvelle version :

1
Génère un lien d'activation sécurisé via /api/admin/clients/generate-link (valable 2h, usage unique)
2
Appelle Apps Script doPost avec l'email, le prénom et le lien
3
Apps Script envoie le template reset_mot_de_passe depuis Gmail
✅ La réponse au client est toujours générique ("si un compte existe…"), pour ne pas révéler si l'email est enregistré.
🔑

Variable d'environnement à ajouter sur Vercel

VariableValeurOù l'obtenir
APPS_SCRIPT_URL URL du Web App Apps Script déployé Apps Script → Déployer → Gérer les déploiements → URL du Web App
⚠️ Sans cette variable, le flux "mot de passe oublié" ne peut pas appeler Apps Script. Le lien sera généré mais l'email ne partira pas. Le flux "accès après paiement" (flux 1) fonctionne déjà indépendamment car Apps Script appelle lui-même l'API.

Tester : cocher is_paiement_recu = oui pour un prospect test → vérifier que l'email part avec un vrai lien cliquable.
