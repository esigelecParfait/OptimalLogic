Mise à jour Apps Script — Liens d'activation client

Ce document décrit les modifications apportées au script Google Apps Script et les deux templates d'email à créer.

5 modifications 2 templates à créer 1 nouvelle variable

1 Contexte

Le script a été mis à jour pour automatiser l'envoi des liens d'activation aux clients :

  • Nouveau client (paiement reçu) → le script génère un lien sécurisé et envoie l'email automatiquement
  • Mot de passe oublié → le site envoie une requête au script qui envoie l'email automatiquement
ℹ️ Le backup du code original est dans docs/google-apps-script-BACKUP-ORIGINAL.js. En cas de problème, recollez ce code dans Apps Script pour revenir à l'état précédent.

2 Modifications du code

Le fichier à utiliser est docs/google-apps-script-UPDATED.js. Voici exactement ce qui a changé :

AJOUT setConfig() — Ligne 85 docs/google-apps-script-UPDATED.js · L.85

Ajout de l'URL du site pour que le script sache où appeler notre API.

// Dans setConfig(), à la fin de properties.setProperties({...}) : APP_URL: "https://optimal-logic.com", // ← LIGNE AJOUTÉE
AJOUT getConfig() — Ligne 100 docs/google-apps-script-UPDATED.js · L.100

Ajout de la récupération de cette URL dans getConfig().

// Dans getConfig(), à la fin du return {...} : appUrl: properties.getProperty("APP_URL"), // ← LIGNE AJOUTÉE
AJOUT replaceVariables() — Ligne 278 docs/google-apps-script-UPDATED.js · L.278

Ajout de la variable {{activation_link}} pour qu'elle soit remplacée dans les templates email.

// Dans replaceVariables(), à la fin de la chaîne de .replaceAll() : .replaceAll("{{activation_link}}", row.activation_link || ""); // ← LIGNE AJOUTÉE
MODIFIÉE maybeSendIdentifiantEmail() — Lignes 396–430 docs/google-apps-script-UPDATED.js · L.396

La fonction appelle maintenant notre API pour générer un lien d'activation sécurisé avant d'envoyer l'email. Ce lien est inséré dans la variable row.activation_link et est valable 2 heures.

// Bloc ajouté au début de maybeSendIdentifiantEmail() : const config = getConfig(); const response = UrlFetchApp.fetch( `${config.appUrl}/api/admin/clients/generate-link`, { method: "post", headers: { "Content-Type": "application/json", "x-admin-secret": config.apiSecret }, payload: JSON.stringify({ email: row.email }) } ); const data = JSON.parse(response.getContentText()); row.activation_link = data.link; // ← injecté dans le template
NOUVELLE FONCTION doPost() — Lignes 432–458 docs/google-apps-script-UPDATED.js · L.432

Nouvelle fonction qui reçoit les demandes de "mot de passe oublié" depuis le site web. Quand un client clique "Mot de passe oublié", le site envoie une requête à cette fonction qui déclenche l'email automatiquement.

function doPost(e) { const data = JSON.parse(e.postData.contents); // Vérifie le secret, puis envoie le template "reset_mot_de_passe" const row = { email: data.email, prenom: data.prenom, activation_link: data.link, // ← lien généré par le site demande_id: "reset_mdp" }; sendTemplateEmail(row, "reset_mot_de_passe"); }

3 La variable à utiliser dans les emails

{{activation_link}}

C'est le lien sur lequel le client clique pour créer ou réinitialiser son mot de passe. Il est généré automatiquement par le site et expire après 2 heures.

Exemple d'utilisation dans un bouton HTML :

<a href="{{activation_link}}" style="display:inline-block;background:#fff;color:#000;padding:14px 28px; border-radius:10px;font-weight:700;text-decoration:none;"> Créer mon mot de passe → </a>
✅ Toutes les autres variables habituelles fonctionnent toujours : {{prenom}}, {{nom}}, {{email}}, {{offre}}, etc.
