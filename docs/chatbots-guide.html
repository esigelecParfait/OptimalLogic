2. Étapes de mise en place du chatbot

Ce qui a été réalisé pour implémenter les chatbots

Voici les 6 étapes concrètes qui ont permis de mettre en place les deux assistants IA sur le site OptimalLogic.

1
Installation du SDK Anthropic
npm install @anthropic-ai/sdk
2
Ajout de la clé API

Dans le fichier .env.local à la racine du projet :

ANTHROPIC_API_KEY=sk-ant-...

La clé ne doit jamais être commitée sur Git. Redémarrer le serveur après modification.

3
Création des routes API (le cerveau)
app/api/chat/client/route.ts → reçoit les messages, charge les données client depuis Supabase, appelle Claude app/api/chat/public/route.ts → même principe, connaît les offres et tarifs du site
4
Création des interfaces UI
app/espace-client/support/ChatInterface.tsx → composant de chat espace client components/ChatWidget.tsx → widget flottant site public
5
Intégration dans le site

La page /espace-client/support affiche la ChatInterface.
Le ChatWidget est ajouté dans app/layout.tsx — visible sur toutes les pages publiques, masqué automatiquement sur /espace-client.

6
Redémarrage du serveur
npm run dev

Aucune base de données supplémentaire, aucun service tiers. Tout passe par l'API Anthropic directement depuis Next.js.

3. Comment fonctionne le chatbot (pour le collaborateur)

Architecture en 3 parties

Chaque chatbot est composé d'une route API côté serveur, d'un composant React côté client, et d'un system prompt qui définit le comportement de l'IA.

1
La clé API Anthropic

Elle est stockée dans le fichier .env.local à la racine du projet, sous la variable ANTHROPIC_API_KEY. Elle ne doit jamais être commitée sur Git ni partagée. Le serveur doit être redémarré après toute modification de ce fichier.

2
Les routes API (backend)

Deux fichiers gèrent les appels à Claude :

app/api/chat/client/route.ts → chatbot espace client app/api/chat/public/route.ts → chatbot site public

Ces routes reçoivent les messages du client, construisent le system prompt avec les données Supabase (pour le chatbot client), puis appellent l'API Anthropic en mode streaming. Elles retournent la réponse token par token (Server-Sent Events).

3
Le system prompt — comment modifier le comportement

Le comportement de l'IA est entièrement défini par le texte du system prompt dans chaque route. Pour modifier ce que le chatbot dit ou ne dit pas :

  • Ouvrir le fichier de route correspondant
  • Localiser la fonction buildSystemPrompt() (chatbot client) ou la constante SYSTEM_PROMPT (chatbot public)
  • Modifier le texte en français directement — pas de code à toucher
  • Sauvegarder ; Next.js recharge automatiquement en mode dev
4
Les composants UI (frontend)

L'interface de chat est dans deux fichiers :

app/espace-client/support/ChatInterface.tsx → espace client components/ChatWidget.tsx → widget public

Ces composants gèrent l'affichage, la saisie, le streaming et le rendu des boutons de choix. Pour changer le design, modifier les classes Tailwind dans ces fichiers.

5
Modèle IA utilisé

Les deux chatbots utilisent Claude Haiku 4.5 (claude-haiku-4-5-20251001), le modèle le plus rapide et économique d'Anthropic. Pour changer de modèle, remplacer cette valeur dans le fichier de route. Coût estimé : moins de 5 €/mois pour un usage normal.

6
Règles de sécurité intégrées

Les règles de sécurité sont inscrites directement dans le system prompt (section "RÈGLES ABSOLUES"). Elles interdisent à l'IA de révéler des informations sensibles, du code source, des données clients, ou de répondre à des questions d'exploitation. Ces règles ne peuvent pas être contournées par l'utilisateur via le chat.

Pour tester : lancer le serveur avec npm run dev, ouvrir http://localhost:3000 pour le widget public, ou se connecter à l'espace client pour tester le chatbot de support.
