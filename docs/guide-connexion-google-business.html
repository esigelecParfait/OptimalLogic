Connecter Google Business Profile
à l'espace client OptimalLogic

Ce guide vous permet de relier les fiches Google Business de vos clients à leur tableau de bord. Une fois configuré, les métriques (vues, clics, avis, note) se mettent à jour automatiquement chaque lundi.

Avant de commencer : assurez-vous d'avoir accès au compte Google qui gère les fiches de vos clients (Google Business Profile Manager), et que l'application OptimalLogic est déployée sur Vercel avec les variables d'environnement accessibles.

1 Créer un projet Google Cloud

1

Aller sur Google Cloud Console

Connectez-vous avec le compte Google qui gère les fiches Business de vos clients.

https://console.cloud.google.com
2

Créer un nouveau projet

Cliquez sur le sélecteur de projet en haut → "Nouveau projet" → nommez-le OptimalLogic → Créer.

3

Activer les deux APIs nécessaires

Allez dans APIs et services → Bibliothèque, recherchez et activez :

  • Business Profile Performance API — pour les vues et clics
  • My Business Account Management API — pour les avis et la note

2 Créer les identifiants OAuth 2.0

1

Configurer l'écran de consentement OAuth

APIs et services → Écran de consentement OAuth → Choisir "Externe" → Remplir : Nom de l'application = OptimalLogic, email d'assistance = votre email → Enregistrer.

2

Créer les identifiants

APIs et services → Identifiants → Créer des identifiants → ID client OAuth 2.0

Type d'application : Application Web

Nom : OptimalLogic Server

URI de redirection autorisée :

https://votre-domaine.vercel.app/api/admin/google-auth/callback
ℹ️ Remplacez votre-domaine.vercel.app par l'URL réelle de votre déploiement Vercel. Pour les tests en local, ajoutez aussi http://localhost:3000/api/admin/google-auth/callback.
3

Copier les identifiants

Après création, Google affiche votre Client ID et Client Secret. Notez-les — vous en aurez besoin à l'étape suivante.

3 Ajouter les variables dans Vercel

Dans votre dashboard Vercel → Projet OptimalLogic → Settings → Environment Variables, ajoutez :

Variable Valeur Source
GOOGLE_CLIENT_ID Votre Client ID Google Cloud Étape 2, identifiants OAuth
GOOGLE_CLIENT_SECRET Votre Client Secret Google Cloud Étape 2, identifiants OAuth
GOOGLE_REFRESH_TOKEN À obtenir à l'étape suivante Flow OAuth (étape 4)
NEXT_PUBLIC_APP_URL https://votre-domaine.vercel.app URL de votre déploiement Vercel
⚠️ Activez les variables pour tous les environnements (Production, Preview, Development). Redéployez le projet après ajout.

4 Obtenir le Refresh Token (autorisation Google)

Cette étape se fait une seule fois. Elle autorise OptimalLogic à lire les données Google Business de votre compte.

1

Lancer le flow d'autorisation

Ouvrez cette URL dans votre navigateur (remplacez par votre ADMIN_SECRET) :

https://votre-domaine.vercel.app/api/admin/google-auth?secret=VOTRE_ADMIN_SECRET
2

Autoriser avec votre compte Google

Google vous demande de vous connecter et d'autoriser l'accès à votre compte Business Profile. Cliquez "Autoriser".

ℹ️ Connectez-vous avec le compte Google qui gère les fiches de vos clients, pas votre compte personnel.
3

Copier le Refresh Token

Vous êtes redirigé vers une page qui affiche votre refresh_token. Copiez-le et ajoutez-le comme variable GOOGLE_REFRESH_TOKEN dans Vercel.

🔒 Ne partagez jamais ce token. Il donne accès en lecture à toutes vos fiches Google Business. Si compromis, révoquez l'accès sur myaccount.google.com/permissions.
4

Redéployer l'application

Dans Vercel → onglet Deployments → cliquez "Redeploy" sur le dernier déploiement pour que les nouvelles variables soient prises en compte.

5 Lier chaque client à sa fiche Google

Pour chaque client, renseignez les identifiants Google Business dans Supabase.

1

Trouver l'identifiant de la fiche (Location ID)

Connectez-vous sur business.google.com → sélectionnez la fiche du client → regardez l'URL :

https://business.google.com/dashboard/l/1234567890123456789/about ↑ c'est le Location ID
2

Trouver l'Account ID

Sur business.google.com, l'Account ID est visible dans l'URL de votre dashboard principal :

https://business.google.com/dashboard/a/123456789/locations ↑ c'est l'Account ID
3

Mettre à jour la table clients dans Supabase

Dans Supabase → Table Editor → clients → trouver le client → renseigner :

Colonne Format Exemple
google_location_name locations/[LocationID] locations/1234567890123456789
google_account_location_name accounts/[AccountID]/locations/[LocationID] accounts/123456789/locations/1234567890123456789

6 Vérifier que tout fonctionne

1

Tester la récupération manuelle des métriques

Déclenchez une mise à jour manuelle en faisant une requête POST :

curl -X POST https://votre-domaine.vercel.app/api/admin/metrics/google \ -H "x-admin-secret: VOTRE_ADMIN_SECRET" \ -H "Content-Type: application/json"

Réponse attendue :

{ "success": true, "updated": 3, "total": 3, "errors": [] }
2

Vérifier dans Supabase

Table client_metrics → les colonnes nb_vues_google, nb_clics_google, nb_avis_google et note_google doivent être remplies pour le mois en cours.

3

Vérifier l'espace client

Connectez-vous avec un compte client → les métriques Google apparaissent dans le tableau de bord.

✅ Si les données s'affichent correctement, la configuration est terminée. Les mises à jour se feront automatiquement chaque lundi à 8h30.

7 Récapitulatif des crons automatiques

Heure Route Ce qu'elle fait Statut
Lundi 8h00 /api/admin/metrics/refresh Agrège RDV (Cal.com), demandes, chatbot depuis Supabase Actif
Lundi 8h30 /api/admin/metrics/google Récupère vues, clics, avis, note depuis Google Business API Après config

✓ Checklist de mise en service

  • Projet Google Cloud créé avec les 2 APIs activées
  • Identifiants OAuth 2.0 créés (Client ID + Secret)
  • URL de callback ajoutée dans les identifiants Google
  • GOOGLE_CLIENT_ID ajouté dans Vercel
  • GOOGLE_CLIENT_SECRET ajouté dans Vercel
  • Flow OAuth lancé via /api/admin/google-auth?secret=...
  • GOOGLE_REFRESH_TOKEN copié et ajouté dans Vercel
  • Application redéployée sur Vercel
  • google_location_name renseigné pour chaque client dans Supabase
  • google_account_location_name renseigné pour chaque client dans Supabase
  • Test manuel effectué via POST /api/admin/metrics/google
  • Métriques visibles dans l'espace client
💡
Problème courant : Si vous obtenez l'erreur "No refresh_token in response", allez sur myaccount.google.com/permissions, révoquez l'accès OptimalLogic, puis relancez le flow OAuth. Google ne renvoie le refresh_token qu'à la première autorisation ou après révocation.
